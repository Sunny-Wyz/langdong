### Q2: 供应商档案管理 - 删除取消时出现 403 / 报错 Bug
- **问题**：在供应商档案管理页面中，点击“删除”按钮唤起二次确认弹窗后，若点击“取消”，控制台或页面会抛出报错（如 403 异常或 Uncaught Promise Error）。
- **解决方案**：这是由于前端 Vue 组件 `SupplierProfile.vue` 中的 `handleDelete` 方法使用了 `this.$confirm` 触发确认框，但没有书写 `.catch()` 回调来捕获用户的取消操作（Promise Rejection）。框架的全局异常拦截器捕获到了这个未处理的 Rejection 并可能被误判或抛出了默认的请求异常提示。**修复方式**是为 `$confirm(...).then(...)` 语句块追加 `.catch(() => { this.$message.info('已取消删除') })`，以优雅拦截取消事件。

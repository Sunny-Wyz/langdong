**最后更新**: 2026-03-26  
**文档集成**: 历史源稿已归档至 `docs/archive/legacy_sources/`

## 1. 系统定位

备件管理系统是一个完整的全流程备件库存管理解决方案，覆盖：
- 基础数据管理（备件、货位、设备、供应商档案）
- 仓储管理（入库、上架、库存查询）
- 采购管理（补货建议、询价比价、订单管理）
- 领用管理（申请、审批、出库、安装登记）
- 维修工单（报修、派工、更换记录）
- AI 智能分析（需求预测、安全库存计算、ABC×XYZ 分类）
- 报表看板（KPI、库存分析、消耗趋势、预警中心）

## 2. 核心模块说明

### 2.1 系统管理（UC-SYS）
**定位**：权限与账号管理

| 子模块 | 主要功能 |
|--------|---------|
| 用户管理 | 创建/删除/修改用户，分配角色 |
| 角色管理 | 创建角色，分配菜单权限 |

### 2.2 基础数据管理（UC-BASE）
**定位**：系统字典数据维护，所有业务依赖

| 子模块 | 主要功能 | 关键字段 |
|--------|---------|---------|
| 备件档案 | 备件编号、名称、价格、安全库存 | `spare_part.code` |
| 货位档案 | 仓库位置、容量 | `location.code` |
| 设备档案 | 设备编码、型号、关联备件配套 | `equipment.code` |
| 供应商档案 | 供应商信息、关联品类 | `supplier.code` |
| 品类字典 | 备件分类标签 | `spare_part_category` |

### 2.3 仓储管理（UC-WH）
**定位**：库存从进门到出门的全过程

| 子模块 | 流程 | 关键操作 |
|--------|------|--------|
| 入库收货 | 采购订单验收 → 入库登记 | 扫单、登记实收数量 |
| 货位上架 | 分配具体货位、更新库存 | 批次管理（FIFO） |
| 库存台账 | 随时查询库存分布 | 总库存、货位分布查询 |

**FIFO 特性** ✅（已实现）：
- 自动按 `in_time` 升序扣减批次
- 完整批次追溯（见 IMPLEMENTATION/FIFO.md）

### 2.4 领用管理（UC-REQ）
**定位**：备件申请申批出库的完整流程

| 子模块 | 功能 | 产生的数据 |
|--------|------|----------|
| 发起申请 | 员工填写需求 | `biz_requisition` |
| 审批申请 | 主管批准/拒绝 | 审批历史 |
| 出库确认 | 仓库发出、库存扣减 | FIFO 批次追溯 |
| 安装登记 | 记录安装设备、时间 | `biz_requisition_item.install_loc` |

**数据流出口**：历史消耗 → AI 预测、ABC×XYZ 分类

### 2.5 维修工单（UC-WO）
**定位**：设备故障维修全流程追踪

| 子模块 | 功能 |
|--------|------|
| 故障报修 | 创建工单，记录故障描述/紧急度 |
| 在线派工 | 分配维修人员 |
| 维修记录 | 记录维修步骤、更换备件 |
| 完工确认 | 记录工时/费用，关闭工单 |

**与其他模块联动**：
- 维修需备件 → 触发领用申请
- 维修费用 → 汇入报表费用分析

### 2.6 采购管理（UC-PO）
**定位**：从需求识别到订单验收的完整采购链

```
智能补货建议
     ↓
发起采购申请
     ↓
供应商询价比价
     ↓
采购订单管理
     ↓
到货验收 → 触发仓储入库
```

**智能补货数据来源**：
- 当前库存（仓储管理）
- 安全库存/补货点（AI 分类计算）

### 2.7 备件智能分类（UC-CLS）
**定位**：ABC×XYZ 自动分类，指导库存策略

```
月度消耗数据 (领用管理)
     ↓
ABC分类 (金额维度) × XYZ分类 (规律维度)
     ↓
安全库存(SS) / 补货点(ROP) 计算
     ↓
采购建议 / AI预测特征
```

**分类矩阵** (9宫格)：
- A：高金额 | B：中金额 | C：低金额
- X：稳定 | Y：波动 | Z：不规律

### 2.8 AI 智能分析（UC-AI）
**定位**：机器学习预测与库存优化

| 功能 | 输入数据 | 输出 |
|------|---------|------|
| 需求预测 | 历史消耗(12月) | 下月预测量 + 90%置信区间 |
| 安全库存计算 | 预测波动 + 提前期 | SS / ROP |

**算法选择**（见 AI_ALGORITHMS/SUMMARY.md）：
- RF（随机森林）：规律型需求 (≥4个月数据)
- SBA（平滑法）：间断型需求 (ADI>1.32)
- FALLBACK（兜底）：数据不足 (<3个月)

### 2.9 报表与看板（UC-RPT）
**定位**：全系统数据的统一决策视图

| 看板/报表 | 数据来源 | 用途 |
|----------|---------|------|
| 管理层看板 | 全系统聚合 | 核心 KPI 一页总览 |
| 库存分析 | 仓储 + 分类 | ABC 分布、滞销识别 |
| 消耗趋势 | 领用管理 | 历史月度趋势、TOP10 |
| 供应商绩效 | 采购管理 | 交货率、合格率对标 |
| 维修费用 | 维修工单 | 按月/按设备的费用拆分 |
| 预警中心 | 全系统 | 库存不足、临期备件预警 |

## 3. 数据流向总览

```
系统管理
  └─────── 权限控制 ──→ 所有模块

基础数据管理
  └─────── 字典数据 ──→ 仓储 / 领用 / 维修 / 采购

采购管理 (到货验收)
  └───────────────→ 仓储管理 (入库)
                      ├─→ 库存台账
                      └─→ 库存数据 ──→ 报表

仓储管理 (库存)
  └───────────────→ 领用管理 (出库)
                      ├─→ 消耗数据 ──→ AI预测
                      ├─→ 消耗数据 ──→ ABC×XYZ分类
                      └─→ 消耗数据 ──→ 报表

领用管理 (出库)
  └───────────────→ 报表 (消耗分析)

维修工单
  ├─→ 领用管理 (需要备件)
  └─→ 报表 (维修费用)

ABC×XYZ分类
  ├─→ 采购管理 (安全库存/补货点)
  └─→ AI预测 (特征数据)

AI预测
  └───────────────→ 采购管理 (预测补货量)

全部模块数据
  └───────────────→ 报表与看板 (汇总展示)
```

## 5. 关键设计特性

### 5.1 FIFO 出库 ✅
- 首先进的批次首先出，确保新鲜度
- 完整批次追溯：每次出库可查到来自哪些批次
- 自动跳过临期/质量问题批次

### 5.2 多维库存管理
- 总库存（spare_part_stock）
- 批次库存（stock_in_item.remaining_qty）
- 货位库存（location 维度分布）

### 5.3 智能补货
- 自动计算安全库存 (SS)
- 自动计算补货触发点 (ROP)
- 当库存≤ROP 时自动触发补货建议

### 5.4 完整溯源
- 出库批次追溯（biz_outbound_batch_trace）
- 工单备件使用记录
- 维修费用明细

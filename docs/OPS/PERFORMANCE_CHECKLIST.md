# 性能基线与优化检查清单

最后更新: 2026-03-26

---

## 1. 关键性能指标

| 指标 | 目标阈值 |
|---|---|
| API P95 | <= 500ms |
| API P99 | <= 1000ms |
| 错误率 | < 1% |
| 慢查询 | 单条 > 1s 需排查 |
| CPU 使用率 | 常态 < 70% |
| JVM 堆使用率 | 常态 < 75% |

---

## 2. 关键场景基线

建议对以下接口建立基线:
1. /api/auth/login
2. /api/report/kpi
3. /api/reorder-suggests
4. /api/classify/result
5. /api/ai/forecast/result
6. /api/outbound-trace/requisition-item/{id}

记录项:
- QPS
- 平均响应
- P95/P99
- 错误率

---

## 3. 发布前检查清单

1. SQL 是否命中索引
2. 分页接口是否限制 size 上限
3. 是否存在全表扫描热点
4. 看板聚合接口是否可缓存
5. 连接池配置是否合理
6. JVM 参数是否符合环境资源
7. 日志级别是否避免 debug 常驻

---

## 4. 数据库检查项

1. 查看慢查询日志
2. EXPLAIN 核心 SQL
3. 检查复合索引覆盖
4. 检查高频排序字段索引

示例:
```sql
EXPLAIN SELECT * FROM stock_in_item
WHERE spare_part_id = ? AND remaining_qty > 0
ORDER BY in_time ASC;
```

---

## 5. 应用层检查项

1. 关键聚合接口启用缓存（如 KPI）
2. 批量写入替代循环单条写入
3. 避免 N+1 查询
4. 超时与重试策略可配置
5. 大报表任务建议异步化

---

## 6. 压测建议

压测工具:
- JMeter（项目已有脚本）

压测阶段:
1. 基线压测（当前版本）
2. 变更后压测（新版本）
3. 对比报告（P95、错误率）

验收通过条件:
- 关键接口 P95 不劣化超过 10%
- 错误率维持 < 1%
- 无新增高危慢查询

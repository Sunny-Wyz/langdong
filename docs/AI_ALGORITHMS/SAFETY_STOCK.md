# 安全库存（SS）与补货点（ROP）计算指南

最后更新: 2026-03-26
数据来源: ../archive/legacy_sources/ai5.md, ../archive/legacy_sources/ai6.md, ../archive/legacy_sources/ai7.md

---

## 1. 目标

本文档定义统一的安全库存计算口径，用于：
- 解释动态法与固定法差异
- 提供可复算公式与参数说明
- 给出实验结果与方法选择建议

---

## 2. 核心符号

| 符号 | 含义 | 单位 |
|---|---|---|
| SS | 安全库存 | 件 |
| ROP | 补货点（再订货点） | 件 |
| L | 提前期（Lead Time） | 天 |
| d_bar | 日均需求 | 件/天 |
| sigma_d | 日需求标准差 | 件/天 |
| k | 服务系数（与服务水平对应） | 无量纲 |

服务系数建议：
- A 类: `k = 2.33`（约 99% 服务水平）
- B 类: `k = 1.65`（约 95% 服务水平）
- C 类: `k = 1.28`（约 90% 服务水平）

---

## 3. 统一计算公式

## 3.1 动态法（推荐）

当有预测区间时，采用：

```text
sigma_d = (upper_bound - lower_bound) / (2 * 1.645)
SS = ceil(k * sigma_d * sqrt(L))
ROP = ceil(d_bar * L + SS)
其中 d_bar = predict_qty / 30
```

说明：
- `1.645` 对应 90% 双侧区间的标准正态近似换算
- 该方法可随波动自动调整库存缓冲

## 3.2 固定法（备选）

```text
SS = SS_fixed
ROP = ceil(d_bar * L + SS_fixed)
```

说明：
- 实现简单，易解释
- 对波动变化不敏感，可能过高或过低

---

## 4. 参数获取建议

1. `L`：来自备件或供应商提前期配置
2. `predict_qty`、`upper_bound`、`lower_bound`：来自预测结果
3. `k`：由 ABC 分级决定
4. 当缺少区间时，可临时用历史窗口标准差估算 `sigma_d`

---

## 5. 示例计算（SP20001）

假设：
- `predict_qty = 7`
- `lower_bound = 5`
- `upper_bound = 9`
- `L = 7`
- `k = 2.33`（A 类）

计算：

```text
sigma_d = (9 - 5) / (2 * 1.645) = 1.22
SS = ceil(2.33 * 1.22 * sqrt(7)) = ceil(7.54) = 8
d_bar = 7 / 30 = 0.233
ROP = ceil(0.233 * 7 + 8) = ceil(9.63) = 10
```

结果：
- `SS = 8`
- `ROP = 10`

## 6. 对比实验结果

## 6.1 真实数据窗口（ai5）

样本: SP20001，窗口 2025-04~2025-10

| 方法 | 覆盖率 | 平均 SS | 平均缺口 |
|---|---:|---:|---:|
| 动态法 | 0.00% | 1.57 | 9.38 |
| 固定法(SS=10) | 57.14% | 10.00 | 3.10 |

解读：
- 在该窗口中，动态法参数偏保守，未有效覆盖误差
- 固定法在此样本窗口下更稳

## 6.2 可控造数窗口（ai6/ai7）

样本: SP20001，窗口 2025-04~2025-10

| 方法 | 覆盖率 | 平均 SS | 平均缺口 |
|---|---:|---:|---:|
| 动态法 | 100.00% | 13.86 | 0.00 |
| 固定法(SS=10) | 57.14% | 10.00 | 0.71 |

解读：
- 当波动与区间信息充分时，动态法能显著提升覆盖能力
- 代价是更高的平均安全库存

## 7. 方法选择建议

按优先级执行：
1. 有预测区间 + 质量稳定: 用动态法
2. 数据缺失或模型不稳定: 用固定法兜底
3. 每月复核覆盖率与缺口，动态调参 `k`

建议监控指标：
- 覆盖率（目标按业务设定，如 >= 90%）
- 平均缺口
- 平均库存占用
- 缺货事件数

---

## 8. 落地校验清单

上线前检查：
1. 预测接口返回上下界不为空
2. 提前期配置不为 0
3. ABC 分类结果可用
4. SS/ROP 结果有上限保护（避免异常值）

运行中检查：
1. 月度回测覆盖率
2. 异常波动备件名单
3. 触发补货与实际缺货事件是否一致
